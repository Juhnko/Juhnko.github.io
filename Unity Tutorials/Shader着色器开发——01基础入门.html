Shader着色器开发——01基础入门

Unity中的GameObject（游戏物体）由Compent（组件）组成。

GameObject（游戏物体）一般有两大Compent组件：
  •     Mesh Filter（网格过滤器）: 存储一个Mesh（网格，模型的网格，就是模型由哪些三角面组成，组成一个什么样子的模型，三角面的一些顶点信息。）
  •     Mesh Renderer（网格渲染器）: 用来渲染一个模型的外观，按照mesh给皮肤，颜色。
    •     Mesh Renderer 是通过 Material（材质）控制模型渲染的样子。  Material（材质）= Shader（着色器）+ Map（贴图）
      • shader：材质特性，反光/光滑。Map：表面颜色/纹理。
 本课程目的可达到：了解基本使用Shader。

1、什么是OpenGL、DirectX 以及 GLSL、HLSL、CG?

    一般大型游戏都需要DirctX。
    图像应用编程接口。都是用于图形渲染的。

    CPU (处理器) 、GPU（显卡）、应用程序（C#）、图形接口（OpenGL、DirectX）
    Shader可以认为是一种渲染命令，有OpenGL或 Direct进行解析，来控制渲染丰富多彩的图形。
    移动使用OpenGL，桌面端一般使用Direct。
    GLSL shader语言用于OpenGL。（Khronos）
    HLSL shader语言用于DirectX。（微软）
    CG（Nvidia英伟达语言）跨平台

2、Unity Shader的分类

  Unity使用的是 ShaderLab （对CG封装了一层）来编写Unity中的Shader。

  编程语言和脚本语言的区别？

  1、 Surface  Shader(表面作色器)                                   Unity独有的着色器，封装了顶点/片元着色器（光源多的情况下使用较好）
  2、Vertex/Fragment Shader（顶点/片元 着色器）   基本的着色器，控制实现效果更多
  3、Fixed Function Shader（固定函数着色器）          
shader "LJK/01 FirstShader"//这里不需要文件名保持一致
{
Properties //属性
{
_Color("颜色",Color)=(1,0,0,1)// _属性名（显示出来的属性名，类型）= 默认的参数值，不需要new
_Vector("向量",Vector)=(1,2,3,4)// 四维向量 float4
_Int("整数",Int)=1234 //float
_Float("小数",Float)=3.14//float
_Range("范围",Range(0,10))=6//范围 float
_2D("图片",2D)="white"{}//默认白色图片 sampler2D
_Cube("天空盒子纹理",Cube)="white"{} //samplerCube
_3D("3D纹理",3D)="black"{}//sampler3D
}
SubShader //子Shader，SubShader可以写多个。（增加Shader的适应能力）
//如果第一个SubShader里面的效果都可以实现，那么就使用第一个SubShader。
//如果显示在这个SubShader里面某些效果实现不了，那么会自动运行下一个SubShader。
{
Pass //至少又一个Pass。在这里编写Shader代码，可以使用CG,也可以使用 HLSLPROGRAM
{
CGPROGRAM
//使用CG语言编写Shader代码
//核心Shader代码
//编程代码需要分号结束
fixed _Color; //float/half/fixed相互代替
// float:二进制32位进行存储 最大
// half: 二进制16位进行存储 -6万～6万
// fixed: 二进制11位进行存储 -2～2 最小，一般颜色都用fixed
float4 _Vector;
float _Int;
float _Float;
float _Rang;
sampler2D _2D;
samplerCUBE _Cube;
sampler3D _3D;
ENDCG
}
}
Fallback "Diffuse" //后备方案
}
Shader "LJK/02 testRunShader"
{
//属性可以没有，但必须又一个SubShader和一个pass块
Subshader
{
pass
{
CGPROGRAM//核心代码
//由系统调用的函数（方法），方法名不固定。参数也不固定。
//顶点函数
#pragma vertex vert//这里声明顶点函数的函数名。
//顶点函数处理顶点。（屏幕上显示的顶点位置）（坐标，信息，相对于自身中心）。
//基本作用：确定位置。从模型空间转到"剪裁空间"（屏幕空间）。
//片元函数
#pragma fragment frag//这里声明片元函数的函数名
//片元对应像素。(屏幕上显示的颜色)
//基本作用：返回模型对应的屏幕上的每一个像素点的颜色值。
//具有返回值。
float4 vert(float4 v:POSITION):SV_POSITION //每个顶点坐标都会调用这个方法
//通过语义告诉系统，这个参数是干什么的。
//POSITION告诉系统需要顶点坐标。
//SV_POSITION用来解释返回值，意思是返回值是剪裁空间下的顶点坐标。
{
//坐标系之间的转换需要"矩阵"来进行转换。
float4 pos=mul(UNITY_MATRIX_MVP,v); //使用宏。将矩阵与坐标相乘(x,y,z,w) w一般用于辅助做坐标转换
return pos;//把坐标从模型空间转到剪裁空间。UNITY_MATRIX_MVP是矩阵
}
fixed4 frag():SV_Target
{
return fixed4(1,1,0,1);
}
ENDCG
}
}
Fallback "Diffuse"
}
Shader "LJK/03 testUseStructShader"
{
SubShader
{
Pass
{
CGPROGRAM
// Upgrade NOTE: excluded shader from DX11; has structs without semantics (struct v2f members temp)
#pragma exclude_renderers d3d11
#pragma vertex vert
#pragma fragment frag
#include <UnityShaderUtilities.cginc>
//从应用到顶点
struct a2v
{
float4 vertex:POSITION;//告诉unity把模型空间下的顶点坐标填充给vertex
float3 normal:NORMAL;//告诉unity把模型空间下的法线方向填充给normal
float4 texcoord:TEXCOORD0;//告诉unity把第一套纹理坐标填充给texcoord
};
//
struct v2f
{
//返回值必须有语义。
float4 position:SV_POSITION; // SV_POSITION语义：返回值是剪裁空间下的顶点坐标。
float3 temp:COLOR0; // COLOR0语义：返回值是颜色类型 float4/fixed4
};
//以下两个函数，系统自动调用
//系统给每个顶点调用
v2f vert(a2v v){
v2f f;
f.position=UnityObjectToClipPos(v.vertex);
f.temp=v.normal;
return f;
}
//系统给屏幕上的每个像素点调用，调用次数会比较多
fixed frag(v2f f):SV_Target{
return fixed4(f.temp,1);
}
ENDCG
}
}
FallBack "Diffuse"
}

语义总结：

在顶点函数和片元函数之间传递值

  • 从应用程序传递到顶点函数(a2v)使用的语义
    •   POSITION 顶点坐标（模型空间下的）
    •   NORMAL 法线( 模型空间下)
    •   TANGENT 切线（模型空间）
    •   TEXCOORD0 ~ｎ　纹理坐标 (0-1)
    •   COLOR 顶点颜色
  • 从顶点函数传递到片元函数使用的语义
    • SV_POSITION  剪裁空间中的顶点坐标 （一般是系统直接使用）
    • COLOR0  可以传递一组值 4个
    • COLOR1  可以传递一组值 4个
    • TEXCOORD0~7 传递纹理坐标
  • 从片元函数传递给系统使用的语义
    • SV_Target  颜色值，显示到屏幕上的颜色
