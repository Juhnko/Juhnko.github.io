无参数无返回值的委托的应用案例：
.net 1.0的使用方式：
public delegate void NoReturnNoPara();
NoReturnNoPara method = new NoReturnNoPara(this.DoNothing);
method.Invoke(); // 委托实例化的调用，（具体传不传参数，传什么参数）跟委托的约束要保持一致
private void DoNothing()
{
Console.WriteLine("这里是DoNothing！");
}
.net 2.0的使用方式：
NoReturnNoPara method = new NoReturnNoPara(
delegate(){ // 匿名方法，将要传入的方法直接搬过来，加上delegate，可以把名字去掉！
Console.WriteLine("这里是DoNothing！");
});
.net 3.0以上的使用方式：
NoReturnNoPara method = new NoReturnNoPara(
()=>{ // goes to； Lambda表达式 ()=>表示方法的参数，{ }方法体 （左边是方法的参数列表，右边是方法体）
Console.WriteLine("这里是DoNothing！");
}
);
NoReturnNoPara method = new NoReturnNoPara(
(int id,string name)=>{ // int string 可以省略掉！编译器的语法糖，可以自动推算出来！
Console.WriteLine($"这里是DoNothing！{id}{name}");
}
);
NoReturnNoPara method = new NoReturnNoPara(
(id,name)=>Console.WriteLine($"这里是DoNothing！{id}{name}"));// 如果方法体只有一行，就可以去掉大括号和分号。多行不可以！
NoReturnNoPara method = (id,name) => Console.WriteLine($"这里是DoNothing！{id}{name}") //new NoReturnNoPara也可以去掉！ 委托实例化，将Lambda表达式来给！
以上是Lambda表达式的由来！

 Action和Func：
      框架提供饿两个泛型委托，不用自己声明了。Action无返回值，Func有返回值。

  Action是无返回值的泛型委托。
Action 表示无参，无返回值的委托。 本质： public delegate void Action<in T>(T obj); //in与out的使用！
Action<int,string> 表示有传入参数int,string,无返回值的委托
Action<int,string,bool> 表示有传入参数int,string,bool,无返回值的委托
Action<int,int,int,int> 表示有传入4个int型参数，无返回值的委托
Action至少0个参数，至多16个参数，无返回值。
public void Test<T>(Action<T> action,T p)
{
action(p);
}
private void DoAction（Action action） // 声明一个方法，传入参数类型为action委托的。
{
action.Invoke();
}
this.DoAction(action01/02/03); // 调用的普通方法，只不过传入参数为action委托
Action+Lambda表达式
Action act01=new Action()=>Console.WriteLine("这里是DoNothing");
Action act01=()=>Console.WriteLine("这里是DoNothing"); // 不带参数的Action
Action<string> action02=(s)=>Console.WriteLine($"传入的参数是{s}"); // 带参数的Action
Action<string> action02 = s =>{}; // 如果只有一个参数可以去掉小括号！
Action<string,int>action03=(s,2)=>Console.WriteLine($"传入的参数是{string}{int}"); // 带有两个参数的,无返回值的Action。




Func是带有返回值的泛型委托。
Func<int> 表示无参，返回值为int的委托 本质是：public delegate TResult Func<out TResult>();
Func<object,string,int> 表示传入参数为object, string 返回值为int的委托
Func<object,string,int> 表示传入参数为object, string 返回值为int的委托
Func<T1,T2,,T3,int> 表示传入参数为T1,T2,,T3(泛型)返回值为int的委托
Func至少0个参数，至多16个参数，根据返回值泛型返回。必须有返回值，不可void
public int Test<T1,T2>(Func<T1,T2,int>func,T1 a,T2 b)
{
return func(a, b);
}
Func+Lambda表达式
Func<int> func=new Func<int>( // 无参数，返回值为int的委托
delegate(){ // 匿名方法
return DataTime.Now.Year;
})
Func<int> func02 = new Func<int>(
() => DateTime.Now.Year
);
Func<int> func02 = () => DateTime.Now.Year; // 无参数，返回值为int的Func委托的最终写法 （如果方法只有一行的情况下，return也可以去掉）
Func<int> func02 = () =>{
int a =DateTime.Now.Year+1;
return a;}; // 无参数，方法体有多行的情况下，return不能省略。
Func<int, string> func03 = new Func<int, string>(t => t.ToString()); // 参数为int，返回值为string的Func委托。
Func<int, string> func04 = (t => t.ToString()); // 参数为int，返回值为string的Func委托的最终写法。
